================================================================================
                             AUTOMATIZACIÓN OPENCART
================================================================================
Proyecto: Automatización Web OpenCart
Patrón de Diseño: Screenplay
Framework: Serenity BDD + Cucumber

================================================================================
1. RESUMEN DE IMPLEMENTACIÓN
================================================================================
Se ha desarrollado una solución de automatización de pruebas para el portal 
e-commerce OpenCart. El objetivo principal fue automatizar el flujo crítico 
de compra (End-to-End) aplicando las buenas prácticas del patrón Screenplay 
para garantizar la escalabilidad y legibilidad del código.

Alcance funcional cubierto:
- Navegación a la aplicación.
- Selección de productos y adición al carrito.
- Flujo de Checkout como invitado (Guest Checkout).
- Diligenciamiento de formularios de facturación y envío.
- Confirmación final de la orden.

================================================================================
2. ARQUITECTURA TÉCNICA
================================================================================
La solución se estructuró siguiendo estrictamente la separación de responsabilidades 
del patrón Screenplay:

A. CAPA DE ACTORES Y HABILIDADES
   - Configuración del Actor capaz de navegar la web (BrowseTheWeb).
   - Gestión del ciclo de vida del navegador mediante Serenity.

B. CAPA DE TAREAS (TASKS)
   Se implementaron tareas modulares que representan las acciones de negocio:
   - 'NavigateTo': Responsable de la apertura inicial.
   - 'AddProduct': Encapsula la lógica de búsqueda y selección.
   - 'ProceedToCheckout': Orquesta los pasos del proceso de compra.

C. CAPA DE INTERFAZ DE USUARIO (USER INTERFACE)
   - Mapeo de elementos web utilizando selectores optimizados (ID, CSS, XPath).
   - Separación de locators por páginas (HomePage, CartPage, CheckoutPage).

================================================================================
3. HERRAMIENTAS Y TECNOLOGÍAS
================================================================================
- Lenguaje: Java 17.
- Motor BDD: Cucumber (Gherkin) para la definición de escenarios legibles.
- Orquestador: Serenity BDD para la gestión de drivers y reportes vivos.
- Gestión de Dependencias: Gradle.

================================================================================
4. RESULTADOS DE EJECUCIÓN
================================================================================
- Escenarios Automatizados: 1 (Flujo completo de compra exitosa).
- Reportes: Generación automática de documentación viva (Serenity Reports) 
  con capturas de pantalla paso a paso.
- Trazabilidad: Los pasos de prueba quedaron vinculados a las definiciones 
  de Cucumber, permitiendo una lectura clara del comportamiento del sistema.

================================================================================
5. CONCLUSIÓN
================================================================================
El ejercicio cumple con el objetivo de implementar un script de automatización 
funcional bajo el patrón Screenplay. La estructura base establecida permite 
la ejecución del flujo principal de negocio y sienta las bases para la futura 
inclusión de nuevos escenarios y validaciones adicionales.
================================================================================
